## Logo System

### Primary Logo

**Wordmark + Icon Combination**
- Use for: Primary brand presence, marketing materials, website header
- File: `logo-stacked-combination-mark.png`

### Icon Variations

#### Full Color Icon (Dark Background)
- **Filename:** `icon-color-dark-background.png`
- **Use case:** App icons, social media profiles, dark UI contexts
- **Background:** Navy (#0B1020) or darker

#### Full Color Icon (Light Background)
- **Filename:** `icon-color-light-background.png`
- **Use case:** Light backgrounds, documents, print materials
- **Background:** White or light grays

#### Gradient Icon (Light Background)
- **Filename:** `icon-color-gradient-light-background.png`
- **Use case:** Premium materials, hero sections, special promotions
- **Background:** Light backgrounds for maximum gradient visibility

#### Monochrome Dark
- **Filename:** `icon-dark.png`
- **Use case:** Single-color printing, watermarks, subtle backgrounds
- **Color:** Navy (#0B1020)

#### Monochrome White
- **Filename:** `icon-white.png`
- **Use case:** Dark backgrounds, overlays, inverted contexts
- **Color:** Warm White (#F5EDE6)

### Logo Clear Space

Maintain a minimum clear space around the logo equal to the **height of the "V"** in VaultArena on all sides.

### Minimum Sizes

- **Digital:** 32px height (icon only), 120px width (full logo)
- **Print:** 0.5 inches height (icon only), 1.5 inches width (full logo)

### Logo Do's & Don'ts

#### ‚úÖ DO:
- Use approved logo files only
- Maintain proper clear space
- Use correct color variations for backgrounds
- Scale proportionally

#### ‚ùå DON'T:
- Alter logo proportions or stretch
- Change colors outside approved palette
- Add effects (shadows, glows, outlines)
- Rotate or distort the logo
- Place on busy backgrounds without adequate contrast

### Color Accessibility

- Maintain **WCAG AA** contrast ratio minimum (4.5:1 for body text)
- Orange (#FF6B1A) on Navy (#0B1020): **7.2:1** ‚úÖ
- Warm White (#F5EDE6) on Navy (#0B1020): **12.8:1** ‚úÖ
- Test all color combinations before implementation

### Color Usage Guidelines

**80/15/5 Rule:**
- **80%** Navy (backgrounds, foundations)
- **15%** Warm White (text, content)
- **5%** Orange (accents, CTAs, highlights)

## Typography

### Primary Typeface: **Inter**

**Rationale:** Clean, modern, excellent screen readability, supports multiple weights, tech-forward aesthetic.

- **Weights:** 400 (Regular), 500 (Medium), 600 (Semibold), 700 (Bold), 800 (Extrabold)
- **Use for:** UI, body copy, navigation, all digital applications
- **Fallback:** -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif

### Display Typeface: **Space Grotesk**

**Rationale:** Geometric, confident, perfect for headlines and brand moments.

- **Weights:** 500 (Medium), 600 (Semibold), 700 (Bold)
- **Use for:** Headlines, hero text, brand statements, large display
- **Fallback:** Inter, sans-serif

### Type Scale

| Element | Font | Size | Weight | Line Height | Usage |
|---------|------|------|--------|-------------|-------|
| **H1 Display** | Space Grotesk | 64px / 4rem | 700 | 1.1 | Hero headlines |
| **H1** | Space Grotesk | 48px / 3rem | 700 | 1.2 | Page titles |
| **H2** | Space Grotesk | 36px / 2.25rem | 600 | 1.3 | Section headers |
| **H3** | Space Grotesk | 28px / 1.75rem | 600 | 1.35 | Subsection headers |
| **H4** | Inter | 24px / 1.5rem | 600 | 1.4 | Card titles |
| **H5** | Inter | 20px / 1.25rem | 600 | 1.4 | Small headers |
| **Body Large** | Inter | 18px / 1.125rem | 400 | 1.6 | Intro paragraphs |
| **Body** | Inter | 16px / 1rem | 400 | 1.6 | Standard text |
| **Body Small** | Inter | 14px / 0.875rem | 400 | 1.5 | Secondary text |
| **Caption** | Inter | 12px / 0.75rem | 500 | 1.4 | Labels, captions |
| **Button** | Inter | 16px / 1rem | 600 | 1 | CTA text |

### Typography Guidelines

#### ‚úÖ DO:
- Use Space Grotesk for headlines and brand moments
- Use Inter for UI and body copy
- Maintain consistent line heights for readability
- Use sentence case for most UI elements
- Limit line length to 60-80 characters for body text

#### ‚ùå DON'T:
- Mix more than 2 typeface families
- Use decorative fonts
- Set body text below 14px
- Use ALL CAPS for long copy
- Sacrifice readability for style

## Imagery & Style

### Photography Style

#### Characteristics:
- **Authentic & Diverse:** Real people, global representation
- **Dynamic Energy:** Motion, action, competitive moments
- **Tech-Forward:** Clean compositions, modern settings
- **Warm Lighting:** Natural light, approachable yet premium

#### Subjects:
- Players strategizing on devices
- Diverse groups celebrating wins
- Close-ups of focused competition
- Technology in natural environments
- Global sports moments

#### Treatment:
- **Color Grading:** Warm shadows, slightly desaturated
- **Overlays:** Navy gradient overlays (0-60% opacity) for text readability
- **Crop:** Focus on human moments, leave breathing room
- **Avoid:** Stock-looking, overly staged, cold clinical tech imagery

### Illustration Style

- **Style:** Geometric, clean lines, minimal detail
- **Color:** Brand palette only (Orange, Navy, White)
- **Use:** Icons, UI graphics, explanatory diagrams
- **Mood:** Professional, confident, accessible

## Iconography & Graphics

### Icon Style

#### Characteristics:
- **Stroke-based** icons (not filled) for UI
- **2px stroke weight** at default size
- **Rounded corners** (2-4px radius)
- **24x24px** base grid
- **Geometric & balanced**

#### Color Usage:
- **Primary:** Warm White (#F5EDE6) on dark backgrounds
- **Accent:** Orange (#FF6B1A) for active/selected states
- **Disabled:** Gray (#8C8C8C)

### Graphic Elements

#### Arena Pattern
A subtle geometric pattern representing the "arena" concept:
- Hexagonal grid (low opacity: 3-5%)
- Use as: Background texture on navy surfaces
- Scale: Consistent, never overwhelming content

#### Accent Lines
- **1-2px** solid lines in Orange
- Use sparingly for emphasis or section breaks
- Can be used as animated loading indicators

#### Data Visualization
- Use brand colors only
- Orange for primary data
- Success green for positive trends
- Maintain clear contrast and accessibility

## Brand Tone & Voice

### Voice Attributes

**Confident** | **Approachable** | **Intelligent** | **Exciting**

VaultArena speaks with the authority of expertise but the warmth of a teammate. We respect our players' intelligence and never condescend.

### Tone Guidelines

#### Confident, Not Arrogant
‚úÖ "Your strategy. Your arena. Your victory."  
‚ùå "We're the best platform ever created."

#### Approachable, Not Casual
‚úÖ "New to fantasy sports? We've got your back."  
‚ùå "Yo! Check out our sick new features lol"

#### Intelligent, Not Complex
‚úÖ "Decentralized technology ensures every transaction is transparent and secure."  
‚ùå "Our blockchain-agnostic smart contract infrastructure leverages distributed consensus mechanisms..."

#### Exciting, Not Hype
‚úÖ "The arena is live. Time to compete."  
‚ùå "üöÄüöÄüöÄ MOON INCOMING!!! DON'T MISS OUT!!!"

### Writing Guidelines

#### Language:
- Use active voice
- Be direct and clear
- Avoid jargon unless explaining it
- Write in present tense when possible

#### Punctuation:
- Use exclamation marks sparingly (only for genuine excitement)
- Oxford comma: Yes
- Em dash for emphasis ‚Äî like this

#### Terms We Use:
- **Arena** (not platform, site, or app)
- **Players** (not users or customers)
- **Compete** (not play or participate)
- **Strategy** (not luck or chance)
- **Decentralized** (when meaningful, not as buzzword)

## Do's & Don'ts Summary

### Visual Identity

#### ‚úÖ DO:
- Use approved logo variations only
- Maintain clear space around logo
- Follow color palette exactly
- Use Space Grotesk for headlines
- Use Inter for body copy and UI
- Test all designs on dark backgrounds first
- Maintain 80/15/5 color distribution
- Ensure WCAG AA accessibility standards

#### ‚ùå DON'T:
- Alter logo proportions or colors
- Use gradients on small text
- Mix more than 3 colors in one component
- Use orange as a background color
- Set body text below 14px
- Use outdated or modified logo files
- Create new color variations
- Sacrifice readability for aesthetics

### Content & Messaging

#### ‚úÖ DO:
- Write with confidence and clarity
- Use "arena" and "players" terminology
- Be authentic and transparent
- Celebrate community and competition
- Explain technical concepts clearly
- Use active voice
- Be consistent across all channels

#### ‚ùå DON'T:
- Use excessive hype or urgency tactics
- Oversimplify or condescend
- Make promises you can't keep
- Use technical jargon without explanation
- Copy competitor messaging
- Use slang or overly casual language
- Overuse exclamation marks
